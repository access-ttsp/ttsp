- миграции делаются только вверх, откат не нужен
- есть функция `migrate`, которая принимает на вход тип миграции (postgres | sqlite), строку подключения и папку с миграциями
- в папке миграций лежат SQL файлы миграций, названные в формате `1-migration-one-comment.sql`
- в зависимости от типа миграции создается соответствующий SQL клиент
- создается системная таблица, в которой указываются уже сделанные миграции по номеру миграции без комментария
- при вызове функции проверяется наличие системной таблицы — она создается при необходимости
- файлы из папки читаются при помощи bun glob
- для чтения контента файлов используется bun file
- не применённые миграции накатываются
- применение миграции и обновление системной таблицы должно быть в транзакции
